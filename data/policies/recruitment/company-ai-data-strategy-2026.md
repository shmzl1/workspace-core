# 云智数科有限公司 AI 与数据人才战略（2026）

本文件适用于 TalentFlow 课程项目中的虚构企业招聘场景，不包含真实企业经营数据或个人信息。

## 公司业务背景

云智数科有限公司面向制造、零售、物流和金融企业，提供企业级数据智能与 AI 应用平台。主要产品包括企业数据湖与实时数据分析平台、企业知识库与 RAG 问答平台、多 Agent 业务自动化平台，以及大模型统一接入、评测、监控和治理平台。

公司当前正从产品原型阶段进入企业交付阶段。技术建设重点由“验证功能可行”转向“在权限隔离、数据质量、稳定性、可观测性和成本约束下持续交付”。招聘判断应围绕这一阶段目标，不以框架数量或概念熟悉度替代工程能力。

## 当前技术招聘目标

- 建设稳定的企业级多 Agent 工作流，支持状态持久化、人工确认、审计和失败恢复。
- 建设批处理、实时处理和数据湖仓平台，为经营分析、机器学习和 Agent 提供可信数据。
- 建设统一模型网关、Embedding 服务和模型可观测平台。
- 提升模型调用成功率、系统可审计性和故障恢复能力。
- 支持企业数据权限隔离、敏感信息保护和最小必要访问。

## 三个岗位及协作关系

- 数据平台研发部的大数据分析与处理工程师负责建设可信数据底座，保证业务数据可追踪、可验证、可及时使用。
- 智能应用研发部的 Agent 应用开发工程师使用企业数据和制度知识开发可解释的智能应用，将模型能力接入受控业务流程。
- AI 基础设施部的 AI 平台与模型服务工程师负责模型、Embedding、重排序和 Agent 服务的部署、监控、切换、降级及成本治理。

三个岗位共同对交付质量负责，但不能互相替代。数据工程师不承担 Agent 业务决策，Agent 工程师不绕过数据权限或模型网关，AI 平台工程师不替代 HR 或业务负责人作最终招聘决定。

## 当前团队能力缺口

- 缺少能够把 Agent 原型建设为具备状态恢复、权限、审计和安全防护的完整工程能力。
- 缺少同时处理批处理、实时流、湖仓建模、数据质量和故障恢复的数据平台能力。
- 缺少统一适配多家模型接口、部署推理服务、建设可观测体系并处理线上故障的模型平台能力。
- 缺少以运行指标、故障记录和可复核证据推动优化的跨部门协作机制。

## 招聘优先级

1. AI 平台与模型服务工程师：优先补齐统一接入、稳定运行、降级和可观测能力。
2. 大数据分析与处理工程师：同步补齐可信数据底座、实时处理和数据治理能力。
3. Agent 应用开发工程师：在平台与数据能力基础上加速企业级智能应用交付。

优先级用于安排招聘资源，不直接形成候选人评分，也不得作为自动录用或淘汰依据。

## 招聘证据原则

- 项目证据优先于技能关键词。应核查项目目标、本人职责、技术决策、数据规模、部署方式、指标和故障处理记录。
- 能够解释真实约束、失败原因和优化结果的证据，优先于只列出框架名称的描述。
- 团队项目必须区分候选人本人完成的部分；无法确认时标记为 `UNKNOWN` 并建议追问。
- 模型推断不得写成简历事实，关键判断必须关联简历、作品、面试记录或可复核材料。

## 公平与人工决策边界

- 不根据学校、年龄、性别、籍贯、照片、婚育状况进行评分，也不使用其他与岗位能力无关的敏感属性。
- Agent 只能整理事实、匹配岗位要求、提示风险和提出面试建议，不得自动录用或淘汰候选人。
- 确定性评分、事实证据和模型叙述必须分开保存；模型不得修改确定性评分。
- 降级、回退、证据不足和信息冲突必须明确展示，由 HR 与业务面试官完成人工复核和最终决定。
