# Agent 应用开发工程师

## 岗位概述

岗位编号为 `JOB-AGENT-001`，隶属智能应用研发部。该岗位负责企业级 Agent、RAG、工具调用和可审计工作流开发，将模型能力接入受权限控制的业务流程，并保证异常、回退和人工确认状态能够被清晰追踪。

## 主要职责

1. 使用 Python 和 FastAPI 开发 Agent 应用，维护清晰的接口契约、输入校验和错误响应。
2. 开发多 Agent 流程和状态管理，支持任务编排、阶段记录、失败恢复与人工确认。
3. 实现 Tool Calling 和企业系统接入，确保工具调用经过权限校验、Service 边界和审计记录。
4. 建设 RAG、知识检索和来源引用，区分真实检索结果、本地回退和模型生成内容。
5. 处理模型超时、格式异常、限流与不可用情况，并落实回退、权限保护和运行审计。

## 必备技能

- Python：能够组织可维护的业务模块并处理异常和结构化数据。
- FastAPI：能够开发具备校验、鉴权和统一错误响应的服务接口。
- Agent：理解状态、节点、工具和人工确认在业务工作流中的边界。
- RAG：能够完成文档检索、来源组织和无命中处理。
- Tool Calling：能够约束工具输入、权限、超时和返回结构。
- PostgreSQL：能够设计运行状态与审计信息的持久化边界。
- REST API：能够维护稳定接口契约并处理外部服务错误。
- Docker：能够交付可复现的本地运行环境并定位基础环境问题。

## 加分技能

- LangGraph：有状态图、节点恢复或人工节点实践。
- MCP：有受控工具接入和权限隔离经验。
- ChromaDB 或 pgvector：有向量检索与元数据过滤经验。
- 混合检索：能结合关键词与向量召回并解释结果差异。
- 结构化输出：能处理模型输出校验、重试和安全回退。
- OpenTelemetry：能使用日志、指标和 Trace 定位工作流问题。

## 证据要求

- **完整项目经历**：说明项目目标、工作流、知识来源、工具接入和交付形态。
- **本人负责范围**：明确本人完成的节点设计、接口、RAG、权限、部署或排障工作。
- **可说明的结果、指标或故障处理经历**：给出成功率、延迟、检索效果、运行规模或一次完整故障处理过程。

## 面试重点

能力维度：

- Agent 工作流设计；
- RAG 工程；
- 模型故障回退；
- 权限与审计。

场景问题：

1. Agent 调用模型超时后，如何区分模型、网络、配置和工作流状态问题，并设计安全回退？
2. RAG 已导入文档但检索不到结果时，如何检查切分、索引、过滤条件、阈值和查询链路？
3. 如何隔离模型叙述与确定性计算，保证模型不能修改候选人评分？

## 人工复核条件

- Agent 或 RAG 经历只有技术名称，无法对应到实际模块、输入输出和运行结果。
- 团队项目中 Tool 权限、状态管理或审计部分的本人职责无法确认。
- 候选人无法说明模型异常回退与确定性评分隔离方式，或描述与项目材料明显冲突。
