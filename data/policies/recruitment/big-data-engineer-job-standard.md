# 大数据分析与处理工程师岗位标准

## 岗位信息

- 公司：云智数科有限公司
- 岗位编号：JOB-DATA-001
- 岗位名称：大数据分析与处理工程师
- 部门：数据平台研发部
- 最低经验要求：18 个月

## 岗位目标

建设面向企业交付的可信数据底座，为经营分析、机器学习、企业知识库和多 Agent 应用提供稳定、及时、可追踪的数据。候选人需要同时理解数据建模、分布式处理、质量治理和生产故障处理。

## 主要职责

1. 建设业务数据采集、清洗、转换、建模和治理流程。
2. 使用 Spark 等技术开发批处理任务，使用 Flink、Kafka 等技术开发实时流处理任务。
3. 建设数据仓库、数据湖和湖仓一体平台，维护清晰的数据分层和主题域边界。
4. 处理日志、订单、用户行为和设备数据，保证数据口径、时效和可追溯性。
5. 为经营分析、机器学习和 Agent 提供经过校验的可信数据集与数据服务。
6. 处理数据倾斜、任务延迟、重复消费、乱序、迟到数据和数据丢失。
7. 建设数据质量检查、血缘、任务监控、告警和问题闭环机制。
8. 优化大规模 SQL、存储布局和分布式计算任务，说明优化基线与资源成本。

## 必备技能

- Python、SQL：能够开发数据处理逻辑并分析复杂查询执行计划。
- Spark、Flink：理解批处理与流处理差异、状态、容错和资源调度。
- Kafka：理解分区、消费组、Offset、重复消费及至少一次等消费语义。
- ETL、数据仓库：能够设计采集、明细、汇总和应用层的数据链路。
- 数据质量：能够定义完整性、唯一性、一致性、及时性和准确性规则。
- Linux、Git：能够排查任务环境问题并参与规范化协作。

## 加分技能

- Hive、ClickHouse、Trino、Doris：有分析型存储、查询加速或联邦查询经验。
- Iceberg、Hudi、Delta Lake：能够说明表格式、事务、增量处理和小文件治理。
- Airflow：有任务依赖、补数、重跑和失败告警设计经验。
- 实时数仓、湖仓一体：能够说明实时与离线口径一致性及分层方案。
- 数据血缘、数据治理：有元数据、权限、质量责任人和变更影响分析经验。
- CDC、Debezium：理解数据库增量采集、Schema 变更和数据一致性风险。

## 重点考察

- 是否能结合延迟、吞吐、数据量和一致性要求解释批处理与流处理选型。
- 是否处理过热点 Key、分区不均或 Join 导致的数据倾斜，并有优化前后证据。
- 是否理解 Kafka 消费语义，能设计幂等、去重、Offset 提交和失败恢复。
- 是否能设计数据仓库分层、公共指标口径和历史数据处理策略。
- 是否具备数据质量、血缘、权限和变更治理意识。
- 是否能说明任务规模、峰值延迟、资源消耗和性能优化结果。

## 真实业务与故障场景

- 订单 CDC 数据发生重复和乱序时，需要利用业务主键、事件时间、幂等写入和补偿机制恢复一致结果。
- Flink Checkpoint 持续失败时，需要检查状态大小、存储、反压、网络和超时配置，而不是直接关闭容错。
- Spark 大表关联出现长尾任务时，需要定位倾斜 Key，并评估预聚合、广播、加盐和分区调整的适用性。
- 数据口径变更影响 Agent 报告时，需要通过血缘识别下游、执行版本化变更并验证历史数据。

## 证据要求

证据应说明项目目标、本人职责、数据来源、处理规模、延迟或吞吐、质量规则、故障和优化结果。只列出技术名视为弱证据；信息不足时标记 `UNKNOWN` 并通过面试追问。
