# 技术岗位招聘证据规则

本规则适用于云智数科有限公司的 Agent 应用开发、大数据分析与处理、AI 平台与模型服务岗位。

## 证据等级

### A 级证据

生产系统、真实客户场景、线上部署、可说明环境的性能指标或故障处理记录。候选人应能解释本人职责、约束、决策、结果和复盘；仅出现客户名称或生产字样不自动构成 A 级证据。

### B 级证据

完整课程项目、竞赛项目或个人项目，能够说明本人职责、技术决策、接口或数据流程、异常处理和交付结果。团队项目未区分本人贡献时不能评为 B 级。

### C 级证据

只写“熟悉”“掌握”“了解”，没有项目目标、实现细节、本人职责或结果证据。

### D 级证据

只出现课程、培训、证书或工具关键词，无法确认候选人是否实际使用或理解其工程边界。

## 证据使用规则

1. 技能关键词不能直接等同于实际能力，必须关联项目、作品、代码、面试记录或其他可复核材料。
2. 团队项目必须区分候选人本人负责部分，不能把团队整体成果全部归于个人。
3. 指标必须说明基线、数据规模、测试环境、统计口径和候选人采取的动作。
4. 没有证据时标记 `UNKNOWN`，不直接判断为不具备，也不自动淘汰。
5. 模型推断不得写成简历事实；推断、事实和人工评价必须明确区分。
6. 所有关键判断必须关联证据，并保留证据等级、来源和待核实项。
7. 不允许使用学校、年龄、性别、籍贯、照片、婚育状况或其他敏感个人属性参与评分。
8. Agent 只能给出事实整理、风险提示和建议，最终决定由 HR 与业务面试官完成。
9. 降级、回退、证据冲突和缺少信息必须明确展示，不得以默认值掩盖。
10. 不得因为候选人没有使用某个特定框架而直接淘汰，应评价其解决问题的能力和可迁移经验。

## 跨岗位证据关注点

- Agent 应用开发工程师：重点核查工作流状态、工具权限、RAG 来源、模型故障回退、提示词注入防护和人工确认。
- 大数据分析与处理工程师：重点核查数据规模、延迟、消费语义、质量规则、血缘、倾斜处理和恢复方案。
- AI 平台与模型服务工程师：重点核查部署、网关适配、成功率与延迟、版本发布、熔断降级、可观测和成本治理。

框架经历可以作为追问线索，不能替代证据，也不能覆盖确定性岗位要求或人工复核结论。
