
# AI 使用日志

## 7月6日

1. **问了AI：** 项目范围、技术架构、开发环境和Git分支如何设计。
2. **AI回答：** 建议采用FastAPI、Vue、PostgreSQL、Agent/RAG架构，日常使用`dev`分支。
3. **改了：** 确定项目范围、分层架构、双端入口及统一开发流程。
4. **AI做错了：** 初版方案范围过大，环境配置过于复杂，部分端口和路径没有结合实际项目。

## 7月7日

1. **AI：** 四人如何分工，以及数据库需要设计哪些表。
2. **AI回答：** 建议按业务模块分工，并建立员工、招聘、考勤、薪资、权限等数据表。
3. **改了：** 明确成员职责、AI禁飞区和数据库关系，建立Alembic迁移结构。

## 7月8日

1. **AI：** 候选人评分固定81%、面试日历卡住、招聘报告不实时如何解决。
2. **AI回答：** 建议检查前端、API、Service、数据库和算法之间的完整调用链。
3. **改了：** 删除固定评分和Mock统计，接入PostgreSQL实时数据，修复智能排期和招聘报告。

## 7月9日

1. **AI：** 为什么候选人评估失败，并统一页面加载、权限和错误状态。
2. **AI回答：** 建议检查API前缀、JWT、响应格式、编号类型和异常处理。
3. **改了：** 对齐前后端字段，区分401、403、业务错误和网络错误，补充空状态与重试。
4. **AI做错了：** 曾把所有异常统一显示成“无法连接网络”，掩盖了真实接口和权限问题。

## 7月10日

1. **AI：** 招聘评分、面试排期和薪资访问控制算法中各函数的作用。
2. **AI回答：** 解释评分权重、时间交集、冲突检查、权限判断和返回结构。
3. **改了：** 补充空数据、非法权重、时区、面试时长和权限边界处理。

## 7月11日

1. **AI：** 如何重构多Agent招聘流程，并实时展示Agent执行过程。
2. **AI回答：** 建议建立招聘策略、简历解析、岗位匹配、面试评估、决策审查和HR报告节点。
3. **改了：** 重构Agent、Tool、Service调用链，加入Run、事件、SSE和人工复核状态。
4. **AI做错了：** 初版把部分目录和接口骨架描述成已完成能力，并错误描述为数据库持久化，实际Run仍是进程内保存。

## 7月13日

1. **问了AI：** 如何接入LLM、RAG、本地知识库和Agent实时执行面板。
2. **AI回答：** 建议增加模型网关、知识检索、超时机制和本地规则回退。
3. **改了：** 接入API Key，修复Agent数据库连接，建立本地知识库并打通Agent之间的调用。
4. **AI做错了：** 初版过度依赖外部模型，并存在假RAG状态、固定延迟和虚假Agent事件，后续才改为真实状态与降级机制。

## 7月14日

1. **问了AI：** 如何进行交叉代码Review、完善多Agent界面并整理验收日志。
2. **AI回答：** 建议检查业务逻辑、并发、安全、异常处理、复杂度和前后端一致性。
3. **改了：** 打通多Agent通路，重构候选人池和评估界面，修复运行过慢、深色模式和高复杂度函数。
