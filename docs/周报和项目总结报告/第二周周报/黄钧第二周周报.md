## 四、遇到的问题与困难

### 技术问题

#### 1. 知识库、数据库与 Agent Prompt 中的岗位信息不一致

原有招聘知识文档数量较多，部分岗位编号、技能要求和面试维度存在重复或不一致，可能导致知识检索结果和岗位匹配结果出现偏差。

**已采取的解决方案：**
统一公司招聘规则和三个岗位标准，并同步调整数据库种子岗位数据。

#### 2. 多 Agent 调用层级较多，职责边界容易混乱

招聘流程同时涉及 Agent、Tool、Service、核心算法、数据库持久化和 SSE 推送，若调用关系不清晰，容易产生重复评分、状态不同步或模型覆盖算法结果等问题。

**已采取的解决方案：**
重新梳理完整调用链，明确各层职责，并将确定性评分与大模型分析进行隔离。

#### 3. SSE 实时状态与前端节点展示需要保持一致

多 Agent 执行过程中会产生多个节点事件，如果事件名称、运行状态或前端映射关系不统一，可能出现节点长期停留在运行状态或结果无法及时显示的问题。

**已采取的解决方案：**
统一后端事件结构和节点状态，检查前端事件传递及状态更新逻辑，完善等待、运行、完成和异常状态之间的转换关系。

#### 4. 动画与圆角样式相互影响

多 Agent 页面入场动画使用的裁剪层与悬浮高亮层未共享圆角设置，导致卡片悬浮时出现直角高亮。

**已采取的解决方案：**
定位具体组件和样式文件，规划通过圆角继承、统一样式变量和调整溢出裁剪方式解决问题。

**需要帮助或待解决：** 无。

### 非技术问题

本周涉及知识库、数据库、后端 Agent、核心算法、SSE 和前端界面多个层级，功能关联范围较大，联调过程中需要反复确认数据来源和调用边界。

**已采取的措施：**
采用“先统一数据标准，再梳理调用链，最后进行界面联动”的推进方式，并通过流程图、状态划分和代码 Review 降低跨模块修改带来的影响。

## 六、心得体会与学习收获

### 本周学习到的新知识和技能

本周进一步掌握了多 Agent 系统中流程编排、工具调用、业务服务、确定性算法和实时事件推送之间的协作方式。

通过招聘知识库精简与数据统一，更加明确了 RAG 系统中知识质量、字段规范和数据一致性对检索结果的重要影响。

在 Prompt 优化过程中，加深了对结构化输出、证据约束、来源引用和模型能力边界控制的理解。

### 对项目和开发流程的思考

多 Agent 系统并不是简单增加多个模型调用，而是需要明确每个 Agent 的职责、输入输出和协作顺序。

对于招聘评分、候选人排序等具有明确业务规则的功能，应由确定性算法负责；大模型更适合完成知识理解、信息提取、原因解释和报告生成。

前后端实时联动不仅需要完成接口调用，还需要统一节点标识、事件结构、运行状态和异常处理方式，才能形成稳定且具有展示效果的完整系统。

### 自我改进方面

在继续完善功能的同时，将更加重视异常流程、数据为空、网络中断和模型调用失败等边界情况，提高系统的稳定性。
