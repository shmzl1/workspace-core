## 四、遇到的问题与困难

* **技术问题：**

  * 招聘知识库、数据库和Agent Prompt中的信息不一致，影响知识检索和岗位匹配结果。
  * 多Agent流程涉及Agent、Tool、Service、核心算法、数据持久化和SSE推送。
  * 调用层级较多，容易出现职责混乱、重复评分或状态不同步。
  * SSE事件名称、节点状态和前端映射不统一时，可能导致页面状态更新不及时。
  * 多Agent页面的入场动画和悬浮高亮样式存在冲突，部分卡片出现直角高亮。
  * **已尝试的解决方案：** 统一招聘规则、岗位标准和数据库岗位数据；重新梳理完整调用链，明确各层职责，并隔离确定性评分与大模型分析；统一SSE事件结构和节点状态；检查前端事件传递及状态更新逻辑；通过圆角继承、统一样式变量和调整裁剪方式解决页面样式问题。
  * **需要帮助或待解决：** 无。
* **非技术问题：**

  * 本周涉及知识库、数据库、后端Agent、核心算法、SSE和前端界面等多个模块，联调时需要反复确认数据来源和调用边界。
  * **已采取的措施：** 按照“先统一数据标准，再梳理调用链，最后完成界面联动”的顺序推进，并通过流程图、状态划分和代码Review减少跨模块修改带来的影响。

## 六、心得体会与学习收获

### 本周学习到的新知识和技能

本周进一步掌握了多 Agent 系统中流程编排、工具调用、业务服务、确定性算法和实时事件推送之间的协作方式。

通过招聘知识库精简与数据统一，明确了 RAG 系统中知识质量、字段规范和数据一致性对检索结果的重要影响。

在 Prompt 优化过程中，加深了对结构化输出、证据约束、来源引用和模型能力边界控制的理解。

### 对项目和开发流程的思考

多 Agent 系统的设计中，最重要的是明确每个 Agent 的职责、输入输出和协作顺序，先明确了这些，后面才能进行功能化模块化分隔。

对于招聘评分、候选人排序等具有明确业务规则的功能，应由确定性算法负责；大模型更适合完成知识理解、信息提取、原因解释和报告生成。

前后端实时联动不仅需要完成接口调用，还需要统一节点标识、事件结构、运行状态和异常处理方式，才能形成稳定且具有展示效果的完整系统。

### 自我改进方面

在继续完善功能的同时，要更加重视异常流程、数据为空、网络中断和模型调用失败等边界情况，提高系统的稳定性。
