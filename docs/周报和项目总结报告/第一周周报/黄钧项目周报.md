### 2. 对项目和开发流程的思考

招聘智能系统不能只展示最终评分，还应展示评分依据、Agent 执行过程、风险提示和最终决策来源。核心算法应保持独立、可测试和可解释，Agent 主要负责流程调度，HR 保留最终决策权。

在联调过程中，应优先统一接口契约和数据来源，避免前端固定数据、后端模拟数据和数据库真实数据同时存在，从而降低后续维护和排查难度。

git提交时需要做好记录，方便后续有需要时回滚。

下周将进一步加强异常场景和边界情况的验证，完善节点失败、事件中断和数据缺失情况下的处理逻辑。同时继续优化多 Agent 看板的展示效果，使系统能体现技术深度。
