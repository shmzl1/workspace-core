### 详情

**BUG-S1-001** — AppShell 导入缺失 mock 文件导致编译失败
- **环境**: Win11 / Vite dev
- **复现步骤**: `npm run dev` 启动前端
- **预期结果**: 编译成功
- **实际结果**: `../mock/recruitmentDashboard` 模块不存在，Vite 报错
- **优先级**: P0
- **状态**: ✅ 已修复 — 已创建 `frontend/src/mock/` 目录及所有缺失数据文件
- **回归结果**: 待验证

**BUG-S1-002** — 各视图使用硬编码静态数据，未接入共享 API 层
- **环境**: 全部视图文件
- **复现步骤**: 打开任意业务页（考勤/薪资/候选人/面试/审计）
- **预期结果**: 数据来自 API 调用（开发阶段可为 mock），页面展示 loading → data/empty/error 状态
- **实际结果**: 所有数据硬编码在 `<script setup>` 中，无异步加载流程，无状态切换
- **优先级**: P0
- **状态**: ✅ 已修复 — 五大核心页面已重构（Attendance / Payroll / CandidateDetail / Interviews / Audit）
- **回归结果**: 待验证

**BUG-S1-003** — 共享反馈组件未被使用
- **环境**: `frontend/src/shared/components/feedback/`
- **复现步骤**: 搜索 `EmptyState` / `LoadingState` / `ErrorState` / `PermissionDenied` 的引用
- **预期结果**: 各视图根据不同状态展示对应组件
- **实际结果**: 四个组件已创建但未被任何视图引用（0 imports across views）
- **优先级**: P0
- **状态**: ✅ 已修复 — 五大核心页面已集成全部反馈组件
- **回归结果**: 待验证

**BUG-S1-004** — 候选人评分页缺少权重调整交互
- **环境**: CandidateDetailView.vue
- **复现步骤**: 进入候选人池页面
- **预期结果**: 可以调整各评分维度权重，调整后候选人的加权总分和排序会实时变化
- **实际结果**: 只有固定排序按钮，无法调整评分权重
- **优先级**: P1

- **回归结果**: 待验证

**BUG-S1-005** — 考勤签到/签退按钮无实际交互反馈
- **环境**: AttendanceView.vue
- **复现步骤**: 点击签到或签退按钮
- **预期结果**: 调用 API（或 mock），显示 loading，成功/失败有明确 Toast 反馈，按钮状态更新
- **实际结果**: 按钮有样式但无实际交互逻辑（原版签退按钮固定 disabled）
- **优先级**: P1
- **状态**: ✅ 已修复 — 新增签到/签退逻辑，含 loading 态 + Toast 反馈 + 状态更新
- **回归结果**: 待验证

**BUG-S1-006** — 面试排期页无冲突可视化提示
- **环境**: InterviewsView.vue
- **复现步骤**: 点击智能排期后查看结果

- **实际结果**: 排期建议面板有冲突文字但日历/日程无视觉区分
- **优先级**: P1
- **状态**: ✅ 已修复 — 日历和日程添加冲突样式（红色边框+标签），冲突检测面板高亮
- **回归结果**: 待验证

**BUG-S1-007** — 审计日志无搜索/筛选和展开详情功能
- **环境**: AuditView.vue
- **复现步骤**: 进入审计页面，尝试搜索或点击某条日志
- **预期结果**: 支持按用户/资源搜索，点击行可展开详情
- **实际结果**: 有搜索输入框但无响应式绑定，日志行不可点击展开
- **优先级**: P1
- **状态**: ✅ 已修复 — 新增搜索过滤（按用户名/资源/部门），点击行展开详情面板
- **回归结果**: 待验证

## Sprint 1 回归清单

以下场景必须在 Sprint 1 结束时逐项验证：

### 招聘-评分链路
- [ ] **场景 1a**: HR 打开候选人池 → 页面展示 loading → 加载完成后展示候选人列表
- [ ] **场景 1b**: 调整"项目经历"权重后点击"查看评估依据" → 按钮显示评估中 → 后端保存评分并刷新排序
- [ ] **场景 1c**: 点击"恢复默认"→ 权重恢复 → 排序恢复
- [ ] **场景 1d**: 选择候选人 → 右侧展示评分维度分解（5维柱状图）
- [ ] **场景 1e**: 在"当前阶段"区域选择目标阶段并点击"推进阶段" → 刷新后阶段仍存在
- [ ] **场景 1f**: 点击"安排面试"→ 通过 `application_id` 跳转到面试日历并自动选中申请

### 招聘-排期链路
- [ ] **场景 2a**: HR 打开面试日历 → 展示 loading → 加载完成后展示面试列表
- [ ] **场景 2b**: 点击"智能排期"→ 按钮显示"排期中..."→ 后端返回真实推荐或冲突原因
- [ ] **场景 2c**: 冲突时段在日历中标红，冲突日程标红
- [ ] **场景 2d**: 排期建议面板展示4列信息（时间/面试官/候选人/冲突检测）
- [ ] **场景 2e**: 点击"确认面试安排"→ 按钮显示"保存中..."→ 保存后刷新日历与近期日程

### 考勤链路
- [ ] **场景 3a**: 员工打开考勤页 → loading → 展示当前考勤状态
