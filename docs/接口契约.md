# 接口契约

本文记录规划中的 API 契约与 Sprint 1 已生成的基础路由。不表示接口已完成联调或测试通过。

当前数据库 ORM、首次迁移文件、基础 Service/Repository 和部分只读 API 已建立。数据库迁移、种子导入和接口联调仍需人工在本地环境执行。

## API 前缀

统一前缀：`/api/v1`

## 统一响应结构

```json
{
  "success": true,
  "data": {},
  "error": null,
  "trace_id": "request-trace-id"
}
```

## 错误返回

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误说明"
  },
  "trace_id": "request-trace-id"
}
```

## 身份上下文

- 所有需要权限的接口从统一认证上下文读取当前用户、角色和员工身份。
- Sprint 1 演示身份通过请求头 `X-Demo-Identity` 提供，格式为 `user_id:username:role[:employee_id]`。
- 未提供演示身份时，默认使用 `1:hr_demo:HR_SPECIALIST:1`。
- Web 与小程序共享员工相关 API。
- 小程序不能调用 HR 招聘、薪资预审、审计后台接口。

## Sprint 1 已生成基础路由

- `GET /health`：健康检查，返回统一响应和 `trace_id`。
- `GET /api/v1/auth/me`：返回当前演示身份上下文。
- `GET /api/v1/auth/demo-users`：读取演示用户列表。
- `GET /api/v1/employees`：读取员工列表。
- `GET /api/v1/employees/me`：读取当前演示员工档案。
- `GET /api/v1/employees/me/leave-balance`：读取当前演示员工年假余额。
- `GET /api/v1/recruitment/jobs`：读取岗位列表。
- `GET /api/v1/recruitment/candidates`：读取候选人列表。
- `GET /api/v1/recruitment/jobs/{job_id}/applications`：读取岗位候选人投递与排序字段。
- `GET /api/v1/interviews`：读取已排面试列表。
- `GET /api/v1/interviews/interviewers`：读取面试官列表。
- `GET /api/v1/interviews/meeting-rooms`：读取会议室列表。
- `GET /api/v1/attendance/today`：读取当前演示员工今日考勤。
- `GET /api/v1/attendance/records`：读取当前演示员工近期考勤。
- `GET /api/v1/payroll/me`：读取当前演示员工薪资摘要，必须经过薪资权限 Service。
- `GET /api/v1/payroll/employees/{employee_id}`：读取指定员工薪资摘要，必须经过薪资权限 Service。
- `GET /api/v1/audit/logs`：读取审计日志列表。

## 规划接口分组

- `/auth`：登录、当前身份、角色上下文。
- `/employees/me`：员工本人信息。
- `/attendance`：签到、签退、今日考勤、本月摘要。
- `/leave`：年假余额和请假摘要。
- `/policies`：制度查询和来源展示。
- `/payroll/me`：员工本人薪资摘要查询。
- `/payroll-review`：HR 薪资预审、待 HR 确认、确认与拒绝。
- `/recruitment`：岗位、候选人、评分、比较和流程。
- `/interviews`：面试排期、会议室、面试官和冲突说明。
- `/audit`：敏感访问和操作审计。
- `/agent`：Agent 任务提交、Trace 和结果查询。

## 关键边界

- 员工本人薪资查询必须经过 `salary_access_control` 权限校验。
- 当前 `backend/app/human_only/salary_access_control.py` 尚未由人工负责人提供时，薪资 Service 会拒绝访问，不模拟或绕过算法。
- HR 薪资预审和 HR 薪资确认是不同动作。
- 签到与签退只记录时间事实，不做位置验证。
- Agent 查询必须返回 Trace 摘要，RAG 问答必须返回来源。
- API 层通过 Service 和 Repository 访问数据库，不得在路由中直接编写业务查询。

## Sprint 1 黄钧外层接口

以下接口已补齐 Route、Service、Repository 外层，尚未声明经过运行验证。涉及评分和排期的结果依赖黄钧人工维护的禁飞区算法；禁飞区未接入时返回可解释状态。

| 方法 | 路径 | 用途 | 当前状态 |
| --- | --- | --- | --- |
| `GET` | `/api/v1/recruitment/dashboard` | 招聘评分外层概览 | 已实现外层 |
| `GET` | `/api/v1/recruitment/jobs` | 查询岗位列表 | 已实现外层 |
| `GET` | `/api/v1/recruitment/candidates` | 查询候选人列表 | 已实现外层 |
| `GET` | `/api/v1/recruitment/applications` | 查询候选人申请列表 | 已实现外层 |
| `POST` | `/api/v1/recruitment/applications/{application_id}/score` | 调用评分 Service，Service 尝试调用 `score_resume(...)` | 依赖禁飞区人工算法 |
| `POST` | `/api/v1/interviews/schedule/preview` | 调用排期 Service，Service 尝试调用 `schedule_interview(...)` | 依赖禁飞区人工算法 |
| `GET` | `/api/v1/payroll-review/records` | HR 薪资预审列表展示 | 已实现外层 |
| `GET` | `/api/v1/payroll-review/records/{record_id}` | HR 薪资预审明细与扣款来源展示 | 已实现外层 |

禁飞区未接入时，评分与排期接口返回状态如 `HUMAN_ONLY_ALGORITHM_NOT_READY`，页面应展示“待黄钧人工接入”，不应伪装为已完成算法结果。
# Sprint 1 联调字段补充

- `GET /api/v1/recruitment/applications` 的列表项包含
  `candidate_name` 与 `job_title`，前端以 `application.id` 作为排期选择值。
- `GET /api/v1/interviews/interviewers` 除面试官资源字段外返回
  `employee_name`，用于展示真实面试官姓名。
- `POST /api/v1/interviews/schedule/preview` 的 `application_id`、
  `candidate.candidate_id`、面试官和会议室均来自当前数据库资源；
  Service 会补充已有面试日程用于冲突检测。
- `GET /health` 用于前端页面加载前的后端连通性检查，不属于 `/api/v1` 前缀。
- `GET /api/v1/recruitment/report?time_range=30d|90d|all` 从招聘岗位、候选人和申请记录
  汇总核心指标、招聘漏斗、部门统计、候选人来源和月度趋势。空库返回零值和空数组。
- 招聘评分写回 `candidate_applications.score_breakdown` 时包含 `match_score` 和
  `overall_score`，报告与候选人池从同一评分记录读取数据。
