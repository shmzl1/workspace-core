# 接口契约

本文记录规划中的 API 契约，不表示接口已实现或已验证。

当前数据库 ORM 与首次迁移文件已经建立，但这不代表任何 API 已经完成、可访问或经过联调。

## API 前缀

统一前缀：`/api/v1`

## 统一响应结构

```json
{
  "success": true,
  "data": {},
  "error": null,
  "trace_id": "request-trace-id"
}
```

## 错误返回

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误说明"
  },
  "trace_id": "request-trace-id"
}
```

## 身份上下文

- 所有需要权限的接口从统一认证上下文读取当前用户、角色和员工身份。
- Web 与小程序共享员工相关 API。
- 小程序不能调用 HR 招聘、薪资预审、审计后台接口。

## 规划接口分组

- `/auth`：登录、当前身份、角色上下文。
- `/employees/me`：员工本人信息。
- `/attendance`：签到、签退、今日考勤、本月摘要。
- `/leave`：年假余额和请假摘要。
- `/policies`：制度查询和来源展示。
- `/payroll/me`：员工本人薪资摘要查询。
- `/payroll-review`：HR 薪资预审、待 HR 确认、确认与拒绝。
- `/recruitment`：岗位、候选人、评分、比较和流程。
- `/interviews`：面试排期、会议室、面试官和冲突说明。
- `/audit`：敏感访问和操作审计。
- `/agent`：Agent 任务提交、Trace 和结果查询。

## 关键边界

- 员工本人薪资查询必须经过 `salary_access_control` 权限校验。
- HR 薪资预审和 HR 薪资确认是不同动作。
- 签到与签退只记录时间事实，不做位置验证。
- Agent 查询必须返回 Trace 摘要，RAG 问答必须返回来源。
- API 层后续应通过 Service 和 Repository 访问数据库，不得在路由中直接编写业务查询。

## Sprint 1 黄钧外层接口

以下接口已补齐 Route、Service、Repository 外层，尚未声明经过运行验证。涉及评分和排期的结果依赖黄钧人工维护的禁飞区算法；禁飞区未接入时返回可解释状态。

| 方法 | 路径 | 用途 | 当前状态 |
| --- | --- | --- | --- |
| `GET` | `/api/v1/recruitment/dashboard` | 招聘评分外层概览 | 已实现外层 |
| `GET` | `/api/v1/recruitment/jobs` | 查询岗位列表 | 已实现外层 |
| `GET` | `/api/v1/recruitment/candidates` | 查询候选人列表 | 已实现外层 |
| `GET` | `/api/v1/recruitment/applications` | 查询候选人申请列表 | 已实现外层 |
| `POST` | `/api/v1/recruitment/applications/{application_id}/score` | 调用评分 Service，Service 尝试调用 `score_resume(...)` | 依赖禁飞区人工算法 |
| `POST` | `/api/v1/interviews/schedule/preview` | 调用排期 Service，Service 尝试调用 `schedule_interview(...)` | 依赖禁飞区人工算法 |
| `GET` | `/api/v1/payroll-review/records` | HR 薪资预审列表展示 | 已实现外层 |
| `GET` | `/api/v1/payroll-review/records/{record_id}` | HR 薪资预审明细与扣款来源展示 | 已实现外层 |

禁飞区未接入时，评分与排期接口返回状态如 `HUMAN_ONLY_ALGORITHM_NOT_READY`，页面应展示“待黄钧人工接入”，不应伪装为已完成算法结果。
