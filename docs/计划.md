# TalentFlow 智聘中枢：三 Sprint 开发计划

## Sprint 1：确定性业务核心闭环

### 时间

```text
Day 4 ～ Day 6
```

### Sprint 目标

完成不依赖 LLM 的核心业务闭环，使招聘评分、面试排期、员工考勤、薪资权限和审计均可通过 Web 端实际操作。

Sprint 1 结束时应能演示：

```text
HR 选择岗位
→ 候选人按评分排序
→ 调整权重后排名变化
→ 生成基础面试排期

员工签到
→ 员工签退
→ 查看今日考勤状态

员工查询本人薪资摘要
→ 系统允许

普通员工查询他人薪资
→ 系统拒绝
→ 写入审计日志
```

4. 确保业务 Service 通过公开函数调用两个禁飞区算法，不在其他位置重新实现评分或排期逻辑。

# Sprint 2：Agent、RAG 与薪资预审闭环

## 时间

```text
Day 7 ～ Day 9
```

## Sprint 目标

在 Sprint 1 确定性业务基础上，将 Agent 深度接入招聘、员工服务和薪资预审主流程。

Sprint 2 结束时应能演示：

```text
员工服务 Agent
→ 查询本人年假
→ 查询本人薪资摘要
→ 检索制度知识库
→ 返回结论与来源

招聘决策 Agent
→ 查询岗位和候选人
→ 调用评分 Service
→ 调用排期 Service
→ 返回推荐候选人、排期方案和招聘报告

HR 生成薪资预审
→ 读取月度考勤
→ 生成补贴、扣款和工资预览
→ AI 解释异常
→ 状态保持“待 HR 确认”
→ HR 确认后写入审计
```
