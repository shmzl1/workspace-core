# TalentFlow 三 Sprint 计划

## 项目边界

TalentFlow 智聘中枢面向招聘决策、员工服务、考勤薪资预审与权限审计。系统采用 Vue Web 管理端、微信小程序员工端、Gradio 内部调试台和一套 FastAPI 后端。

## AI 使用边界

AI 可辅助生成工程代码、页面、接口、Service、Repository、Agent、RAG、普通测试和文档。三个 AI 禁飞区及其核心测试由人工负责人完成，AI 不得创建或修改。

## Sprint 1：Day 3-Day 6

### 范围

- 招聘评分与排期最小闭环。
- 签到签退。
- 员工本人薪资查询。
- 权限拒绝与审计。
- 薪资预审数据模型和规则基础。

### 不做

- 真实 LLM。
- RAG。
- 小程序。
- 最终薪资确认。

### 最终效果

形成 Web 管理端与 FastAPI 后端的基础闭环，完成关键禁飞区外层调用路径规划。

## Sprint 2：Day 7-Day 9

### 范围

- 员工服务 Agent、招聘决策 Agent、RAG。
- 薪资预审、AI 异常解释、待 HR 确认和 HR 确认。
- 考勤月度汇总。
- 候选人比较、招聘流程看板、排期冲突解释。

### 不做

- 真实微信 OAuth。
- 银行发薪。
- 税务、社保、公积金。
- Agent 自动确认工资。

### 最终效果

形成 Agent 可解释能力、RAG 来源展示和薪资预审闭环。

## Sprint 3：Day 10-Day 11

### 范围

- 小程序员工端。
- 笔记本局域网 FastAPI。
- Web 与小程序数据一致性。
- Docker、演示数据、部署说明。
- UI、测试、Bug 收口和答辩。

### 不做

- 公网部署。
- 真实企业数据。
- 复杂小程序 HR 功能。

### 最终效果

形成可演示的 Web + 小程序 + FastAPI + Gradio 项目路径。

## 验收场景

- 招聘驾驶舱。
- 候选人评分与解释。
- 权重沙盘与候选人比较。
- 智能排期与冲突说明。
- 员工签到签退。
- HR 薪资预审与待 HR 确认。
- 薪资权限与审计。
- 员工服务 Agent 与 RAG 来源。
- 小程序员工端。
- Gradio 调试台。

## 每日协作规则

- 开发前同步 `dev`。
- 结束前提交清晰变更说明。
- 不直接向 `main` 提交。
- 不提交真实 `.env`、真实数据、真实密钥和本地运行产物。
- 修改需求、接口、数据模型、权限或薪资规则时同步更新文档。

## 最终答辩主线

从“招聘决策可解释”进入，串联“员工服务、考勤事实、薪资预审、权限审计、Agent Trace 与 RAG 来源”，突出一套 FastAPI 后端支撑 Web、小程序和 Gradio 的一致架构。
