**目标：Day 6 前，完成“岗位—筛选—排期—权限”的基础闭环，并能应对 Bug 注入。**

## Sprint 1 最终效果

> HR 选择“Java 后端开发实习生”岗位，系统对候选人进行评分和排序。
>
> HR 调整评分权重后，候选人排名实时变化。
>
> HR 选择候选人后，系统给出基础面试时段。

> 普通员工查询他人薪资时，系统拒绝访问并记录审计日志。

# Sprint 2：Agent 与业务深度融合

**目标：Day 9 前，系统不仅能完成基础业务，还能让 Agent 串起“筛选—排期—报告”和“员工查询—制度检索—权限判断”。**

## Sprint 2 最终效果

> HR 输入：“筛选 Java 后端岗位最合适的三人，并安排他们本周完成技术面。”
>
> Agent 自动读取岗位、候选人、评分结果和面试资源，调用黄钧的评分、排期模块，输出候选人建议与面试方案。
>
> 员工输入：“我还有多少年假，下周请三天是否符合制度？”
>
> Agent 查询余额、检索制度并显示来源依据。
