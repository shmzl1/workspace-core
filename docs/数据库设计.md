## 数据库表项

- `users`：登录账号、密码哈希、角色和持久化权限代码数组。
- `employees`：员工档案和组织信息。
- `leave_balances`：年假余额。
- `attendance_records`：签到、签退和考勤状态。
- `work_calendars`：工作日历和标准工作日。
- `jobs`：岗位信息和岗位画像。
- `candidates`：候选人档案。
- `candidate_pipeline_records`：候选人招聘流程流转。
- `candidate_applications`：候选人与岗位的投递关系、评分结果和当前阶段。
- `interviewers`：面试官。
- `meeting_rooms`：会议室。
- `interview_slots`：可用时间槽。
- `interviews`：面试安排和冲突说明。
- `salary_records`：员工薪资基础记录。
- `payroll_periods`：薪资周期。
- `payroll_rules`：薪资预审规则。
- `payroll_adjustments`：补贴、奖金和扣款调整。
- `payroll_review_records`：薪资预审记录。
- `payroll_line_items`：薪资预审明细项。
- `policy_documents`：制度文档与来源。
- `audit_logs`：审计日志。
- `notifications`：通知和待办。
- `agent_runs`：Run 所有者、状态、State JSON、Snapshot JSON、当前节点和终止信息；`owner_user_id` 使用 `SET NULL` 保留审计历史。
- `agent_run_nodes`：节点状态、attempt、安全输入输出摘要和错误摘要，`(run_id, node_name, attempt)` 唯一。
- `agent_events`：按 `sequence_no` 保存兼容前端的结构化事件，`event_id` 和 `(run_id, sequence_no)` 唯一。
- `agent_tool_calls`：保存 Tool 开始、完成或失败状态及安全摘要，通过事件 ID 关联开始和完成事件。

## Agent Runtime 索引与删除策略

- `agent_runs`：索引 `(owner_user_id, updated_at)` 与 `(status, updated_at)`。
- `agent_run_nodes`：索引 `(run_id, status)`。
- `agent_events`：索引 `(run_id, sequence_no)` 与 `trace_id`。
- `agent_tool_calls`：索引 `(run_id, created_at)` 与 `(run_id, tool_name)`。
- 用户删除不会静默级联删除历史 Run；`owner_user_id` 置空。删除 Run 时，其节点、事件和 Tool 调用级联删除。
- `state_json`、`snapshot_json`、事件摘要和 Tool 摘要使用 PostgreSQL `JSONB`，不保存隐藏思维链、API Key、完整简历、完整联系方式或完整薪资。

## 审计要求

- HR 确认动作记录确认人和确认时间。
- 每一项扣款关联考勤来源与规则来源。
- 薪资访问审计保留角色、目标员工、字段、结果、时间和 `trace_id`。
- payroll 只读取考勤事实和月度汇总，不反向修改原始考勤记录。

## 建模约束

- 主键统一使用自增整数 `id`。
- 金额、天数、数量使用 `Numeric`，避免使用 Float。
- 业务状态使用 `String` + `CheckConstraint`，不使用 PostgreSQL Enum。
- 灵活字段使用 PostgreSQL `JSONB`，迁移中的空数组和空对象使用 PostgreSQL 兼容默认值。
- 时间字段使用 `DateTime(timezone=True)`。
- 通用业务表使用 `created_at`、`updated_at`；日志和流水类表只保留必要时间字段。
- ORM 只表达数据结构和数据库约束，不写业务规则、权限判断或 Agent 逻辑。
- `users.permissions` 使用 PostgreSQL `JSONB` 非空数组，默认 `[]`；权限代码由认证模块统一定义，正常请求从数据库读取而非按用户名推导。
