# 数据库设计

记录核心实体、约束和初始迁移基线。当前已建立 SQLAlchemy ORM 模型与 Alembic 首次迁移文件。

## 当前迁移状态

- 迁移文件：`backend/alembic/versions/0001_initial_schema.py`。
- 覆盖范围：账号、员工、考勤、招聘、面试、薪资预审、制度、审计、通知共 22 张表。
- 当前状态：迁移文件已创建，尚未执行 `alembic upgrade head`。
- 本次未写入种子数据，未创建真实账号、候选人、薪资或企业制度数据。

## 数据库表项

- `users`：登录账号、密码哈希、角色和持久化权限代码数组。
- `employees`：员工档案和组织信息。
- `leave_balances`：年假余额。
- `attendance_records`：签到、签退和考勤状态。
- `work_calendars`：工作日历和标准工作日。
- `jobs`：岗位信息和岗位画像。
- `candidates`：候选人档案。
- `candidate_pipeline_records`：候选人招聘流程流转。
- `candidate_applications`：候选人与岗位的投递关系、评分结果和当前阶段。
- `interviewers`：面试官。
- `meeting_rooms`：会议室。
- `interview_slots`：可用时间槽。
- `interviews`：面试安排和冲突说明。
- `salary_records`：员工薪资基础记录。
- `payroll_periods`：薪资周期。
- `payroll_rules`：薪资预审规则。
- `payroll_adjustments`：补贴、奖金和扣款调整。
- `payroll_review_records`：薪资预审记录。
- `payroll_line_items`：薪资预审明细项。
- `policy_documents`：制度文档与来源。
- `audit_logs`：审计日志。
- `notifications`：通知和待办。

## 审计要求

- HR 确认动作记录确认人和确认时间。
- 每一项扣款关联考勤来源与规则来源。
- 薪资访问审计保留角色、目标员工、字段、结果、时间和 `trace_id`。
- payroll 只读取考勤事实和月度汇总，不反向修改原始考勤记录。

## 建模约束

- 主键统一使用自增整数 `id`。
- 金额、天数、数量使用 `Numeric`，避免使用 Float。
- 业务状态使用 `String` + `CheckConstraint`，不使用 PostgreSQL Enum。
- 灵活字段使用 PostgreSQL `JSONB`，迁移中的空数组和空对象使用 PostgreSQL 兼容默认值。
- 时间字段使用 `DateTime(timezone=True)`。
- 通用业务表使用 `created_at`、`updated_at`；日志和流水类表只保留必要时间字段。
- ORM 只表达数据结构和数据库约束，不写业务规则、权限判断或 Agent 逻辑。
- `users.permissions` 使用 PostgreSQL `JSONB` 非空数组，默认 `[]`；权限代码由认证模块统一定义，正常请求从数据库读取而非按用户名推导。
