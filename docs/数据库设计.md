# 数据库设计

本文记录规划中的核心实体和约束，不表示数据库迁移已完成。

## 核心实体

- `users`：登录账号、角色和认证信息。
- `employees`：员工档案和组织信息。
- `leave_balances`：年假余额。
- `attendance_records`：签到、签退和考勤状态。
- `work_calendars`：工作日历和标准工作日。
- `jobs`：岗位信息和岗位画像。
- `candidates`：候选人档案。
- `candidate_pipeline_records`：候选人招聘流程流转。
- `interviewers`：面试官。
- `meeting_rooms`：会议室。
- `interview_slots`：可用时间槽。
- `interviews`：面试安排和冲突说明。
- `salary_records`：员工薪资基础记录。
- `payroll_periods`：薪资周期。
- `payroll_rules`：薪资预审规则。
- `payroll_adjustments`：补贴、奖金和扣款调整。
- `payroll_review_records`：薪资预审记录。
- `payroll_line_items`：薪资预审明细项。
- `policy_documents`：制度文档与来源。
- `audit_logs`：审计日志。
- `notifications`：通知和待办。

## 考勤状态

- 正常
- 迟到
- 早退
- 缺勤
- 无薪请假
- 已批准年假

## 薪资状态

- 草稿
- 已生成预审
- 待 HR 确认
- 已确认

## 审计要求

- HR 确认动作记录确认人和确认时间。
- 每一项扣款关联考勤来源与规则来源。
- 薪资访问审计保留角色、目标员工、字段、结果、时间和 `trace_id`。
- payroll 只读取考勤事实和月度汇总，不反向修改原始考勤记录。
