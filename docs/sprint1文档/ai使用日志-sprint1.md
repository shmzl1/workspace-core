# TalentFlow 项目 AI 使用日志

**主要 AI 工具：ChatGPT、VS Code Codex**

## 使用原则

1. AI 主要用于需求拆解、架构设计、文档整理、非核心工程代码辅助、界面开发和问题定位。
2. 所有 AI 生成或修改的内容均由组员人工检查后决定是否采用。
3. 以下 AI 禁飞区核心算法及其核心测试不允许由 AI 创建或修改：

   * `backend/app/human_only/resume_scoring.py`
   * `backend/app/human_only/interview_scheduler.py`
   * `backend/app/human_only/salary_access_control.py`
   * 对应的三个 `backend/tests/human_only/` 核心测试文件
4. AI 不负责最终录用、淘汰、排期确认和薪资确认，相关业务决定必须由人工完成。
5. AI 辅助完成工程修改后，数据库迁移、服务启动、功能验收和 Git 提交均由组员人工处理。

## 2026 年 7 月 6 日

**使用目的：项目初始化和工程目录规划。**

向 AI 说明企业人力资源管理 Agent 的项目范围、四人协作方式、前后端技术栈以及 `main`、`dev` 双分支工作流，请 AI协助规划项目目录和模块边界。

AI 辅助整理了以下内容：

* 规划 `backend`、`frontend`、`docs`、`data`、`infra` 和 `scripts` 等顶层目录。
* 将后端划分为招聘、面试、员工、考勤、薪资、制度、审计、Agent、RAG 和 AI 禁飞区等模块。
* 建立前端页面、共享组件、API、路由、状态管理和测试目录。
* 整理 README、协作规范和初始项目说明。
* 明确 `dev` 为开发主线、`main` 为稳定版本分支。

组员对目录结构进行了人工调整，删除不符合当前项目规模的复杂设计后，将初始工程骨架提交到仓库。当天未使用 AI 编写三个禁飞区核心算法。

**对应仓库记录：** 项目仓库初始化、项目内容更新、目录更新及 `dev` 分支合并。

## 2026 年 7 月 7 日

**使用目的：完善项目架构、需求文档、环境配置和前端视觉原型。**

向 AI 提供课程项目要求、最终功能范围和展示需求，请 AI 协助将项目从空目录完善为可以继续开发的模块化工程。

AI 主要辅助完成：

* 整理项目需求说明、架构设计和环境配置文档。
* 明确一套 FastAPI 后端由 Vue Web、微信小程序和 Gradio 调试台共同使用。
* 明确普通业务调用链为
  `API → Service → Repository → PostgreSQL`。
* 明确 Agent 调用链为
  `Agent → Tool → Service → human_only`。
* 明确 RAG、Agent、数据库和前端之间的职责边界。
* 辅助生成 Vue 前端视觉原型，包括招聘看板、候选人详情、筛选流水线、每周日程等页面。
* 辅助排查并修复 `@vitejs/plugin-vue` 版本和前端依赖配置问题。
* 完善 Agent 操作规范，限制 AI 自动执行服务启动、数据库迁移、构建和高风险 Git 命令。

组员人工审核了前端布局、页面文案和架构说明，删除过度设计内容，并根据课程周期调整为模块化单体架构。

 **对应仓库记录：** 架构重构、环境配置、需求文档、前端 UI 设计和 Agent 约束修改。

## 2026 年 7 月 8 日

**使用目的：推进 Sprint 1 功能开发和前后端工程联调。**

向 AI 提供招聘评分、候选人管理、面试排期、考勤、薪资预审和权限审计的接口要求，请 AI 只修改禁飞区以外的工程代码。

AI 辅助完成：

* 建立招聘模块的 Route、Service、Repository 和 Schema 外层。
* 增加岗位、候选人、申请记录和评分接口。
* 建立面试排期预览接口及结构化请求、响应模型。
* 将候选人、面试官、会议室和时间段数据传递给人工排期算法。
* 完善 HR 薪资预审列表、详情、扣款来源和待确认状态展示。
* 补充员工签到、签退、考勤查询和部分审计功能。
* 完善候选人池、面试日历和权重沙盘的前端交互。
* 重构前端共享 API 层、统一布局和通用加载、空状态、错误提示组件。
* 辅助定位分支合并冲突、错误导入、SQLite 测试约束和页面布局退化问题。

简历评分、面试排期和薪资访问控制的核心规则由对应负责人人工编写。AI 只负责其外层调用接口、数据转换、异常处理和页面展示，不生成或模拟核心算法。

当天还进行了多次 `dev` 分支合并和冲突处理，组员人工保留正确的前端接口逻辑和后端新增功能。

**对应仓库记录：** 面试评分和智能排期、手动智能筛选和排期、AI 禁飞区提交、考勤及薪资权限功能、前端架构重构、UI 修复和分支冲突处理。

## 2026 年 7 月 9 日

**使用目的：修复联调问题并将候选人池、面试日历改为动态数据。**

根据实际运行中出现的“智能评估无法获取”“候选人评分固定”“面试日历无法正确选择人员”等问题，向 AI 提供错误现象和相关代码，请 AI 分析禁飞区以外的工程原因。

AI 辅助完成：

* 修复前后端跨域配置和部分请求异常。
* 统一后端异常响应格式、错误代码和 `trace_id`。
* 将数据库端口配置调整为项目实际使用的端口。
* 为候选人池增加岗位下拉选择。
* 根据岗位动态过滤候选人，不再固定展示同一批人员。
* 从岗位、候选人和申请记录接口组合页面数据。
* 将候选人申请编号通过路由参数传递到面试日历。
* 在面试日历中增加候选人选择，并使用真实申请编号生成排期请求。
* 修复候选人池和面试日历之间跳转后数据不一致的问题。
* 调整本地数据仅作为接口异常时的展示兜底，不伪造接口成功状态。

组员人工检查了候选人、岗位和申请记录的对应关系，并结合数据库数据验证页面是否能够随数据变化。

**对应仓库记录：** CORS 与异常格式修复、候选人池和面试日历动态岗位及候选人选择、Sprint 功能整合和调试提交。

## 2026 年 7 月 10 日

**使用目的：完成登录权限体系、Sprint 1 稳定版本整理和招聘多 Agent 架构设计。**

首先向 AI 提供现有演示身份机制的问题，要求将系统调整为真实登录和后端权限校验。

AI 辅助完成：

* 增加用户名和密码登录接口。
* 使用 JWT 标识当前账号。
* 每次业务请求从数据库重新读取账号状态、角色、权限和员工档案。
* 为用户增加持久化权限字段及数据库迁移文件。
* 将考勤、招聘、面试、薪资和审计接口改为权限依赖校验。
* 完善前端登录页面、Token 保存、当前用户信息读取和权限拒绝提示。
* 整理 Sprint 1 功能并形成稳定版本。

随后根据 Sprint 2 规划，使用 AI 辅助设计招聘多 Agent 目录和数据契约：

* 招聘策略 Agent。
* 简历解析 Agent。
* 岗位匹配 Agent。
* 面试评估 Agent。
* 决策审查 Agent。
* HR 最终报告。
* Agent 公共状态、事件、Trace、Guardrail 和模型网关契约。
* Tool、招聘服务、RAG 摄取、检索和引用契约。
* 前端招聘 Agent 执行看板的数据结构。

`main` 分支主要保存 Sprint 1 稳定业务闭环及多 Agent 目录、契约和静态节点骨架；`dev` 分支进一步包含进程内 Agent Run、规则式招聘策略计划、SSE 事件和前端实时看板代码，但仍需本地人工验收。真实 LangGraph、LLM 和 ChromaDB RAG 尚未正式接入。

AI 未获得自动录用、自动淘汰、自动确认排期或自动确认薪资的权限。

**对应仓库记录：** 登录功能、Sprint 1 合并与发布、多 Agent 目录重构。

## 本周期总结

本周期内，AI 主要承担了需求和架构整理、工程目录搭建、非核心代码辅助、Vue 页面开发、接口外层实现、登录权限改造、错误定位以及多 Agent 契约设计等辅助工作。

项目中的简历评分、面试排期和薪资访问控制属于 AI 禁飞区，其核心实现和核心测试由组员人工维护。所有 AI 输出均需经过人工检查，AI 不拥有任何最终招聘、排期或薪资决策权。
