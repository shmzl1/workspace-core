# 演示脚本

本文规划 7-8 分钟演示顺序，不表示所有功能当前已经实现。

## 1. 招聘驾驶舱

- 展示目的：说明 HR 总览招聘状态。
- 关键操作：打开 Web 工作台招聘看板。
- 预期现象：展示 KPI、筛选流水线、日程和热度矩阵。
- 失败兜底：使用静态原型说明后续接入数据。

## 2. 岗位与候选人评分

- 展示目的：说明岗位画像和候选人评分流程。
- 关键操作：进入 Web 工作台 `Sprint 1 闭环`，打开“招聘评分”，点击“生成评分”。
- 预期现象：展示评分维度、候选人差距、评分结果或“禁飞区算法待黄钧人工接入”。
- 失败兜底：说明禁飞区算法由人工负责人解释。

## 3. 权重沙盘与候选人比较

- 展示目的：说明 HR 可调整筛选权重。
- 关键操作：调整技能、项目、到岗时间权重。
- 预期现象：候选人排序和解释变化。
- 失败兜底：展示规划文档中的评分边界。

## 4. 智能排期

- 展示目的：说明候选人、面试官、会议室和时间槽约束。
- 关键操作：进入 `Sprint 1 闭环` 的“智能排期”，点击“生成排期建议”。
- 预期现象：展示推荐时间、面试官、会议室、冲突原因，或“排期算法待黄钧人工接入”。
- 失败兜底：由人工负责人解释排期禁飞区输入输出。

## 5. 员工签到签退

- 展示目的：说明考勤只做时间签到签退。
- 关键操作：员工提交签到或签退。
- 预期现象：今日状态更新。
- 失败兜底：展示规则文档和演示数据。

## 6. HR 薪资预审

- 展示目的：说明考勤进入薪资预审。
- 关键操作：进入 `Sprint 1 闭环` 的“HR 薪资预审”，查看预审列表、扣款来源和规则说明。
- 预期现象：展示待 HR 确认状态。
- 失败兜底：展示薪资预审公式。

## 黄钧 Sprint 1 演示路径

1. 打开 Web 工作台的 `Sprint 1 闭环`。
2. 在“招聘评分”中点击评分入口。
3. 查看评分解释，或查看禁飞区待接入提示。
4. 切换到“智能排期”。
5. 查看排期建议区域和冲突说明。
6. 切换到“HR 薪资预审”。
7. 查看扣款来源、规则说明和待 HR 确认状态。

## 7. 薪资权限与审计

- 展示目的：说明字段脱敏和敏感访问审计。
- 关键操作：用不同角色查看薪资信息。
- 预期现象：无权限访问被拒绝并记录审计。
- 失败兜底：展示权限与审计设计文档。

## 8. 员工服务 Agent 与 RAG 来源

- 展示目的：说明制度问答和来源追踪。
- 关键操作：询问请假或薪资保密制度。
- 预期现象：返回回答、来源和 Trace。
- 失败兜底：展示 Gradio 调试台规划。

## 9. 小程序员工端

- 展示目的：说明员工端轻量入口。
- 关键操作：展示首页、考勤、年假、本人薪资摘要。
- 预期现象：与 Web 员工数据一致。
- 失败兜底：展示小程序接入说明。

## 10. Gradio 调试台

- 展示目的：说明内部 Agent 调试能力。
- 关键操作：查看执行链、工具调用和 RAG 命中内容。
- 预期现象：展示 Trace 与错误信息。
- 失败兜底：使用文档说明调试台定位。
