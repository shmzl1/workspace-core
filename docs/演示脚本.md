# 演示脚本

本文描述最终答辩目标，不表示当前所有功能均已完成或验收通过。

## 1. 企业招聘目标

HR 选择岗位，设置目标人数、必备/加分技能、最低经验、评分阈值、可信度阈值、截止日期和紧急程度。

## 2. 招聘策略 Agent

展示策略节点读取目标、候选人范围并生成结构化执行计划。当前相关代码存在，待本地人工验收；完整总控能力为计划中。

## 3. 简历解析 Agent

展示计划中的候选人画像、缺失项和可定位简历证据，不把无证据信息表述为已确认事实。

## 4. 岗位匹配 Agent

展示计划中的岗位 Rubric、必备条件、确定性评分、技能/项目缺口和企业知识来源。

## 5. 面试评估 Agent

有真实结构化面试数据时展示评价；无数据时明确显示待面试，不生成虚假评价。

## 6. 实时 Agent 执行过程

展示创建 Run、`run_id`、SSE、结构化 `AgentEvent`、节点状态和当前动作。`AGENT_THINKING` 只展示可审计结构化阶段摘要。

## 7. 决策审查

展示无证据高分、必备条件、来源版本、专业节点分歧和人工复核标记；不静默修改确定性评分。

## 8. 候选人比较与可信度

展示候选人比较矩阵、评分与可信度分离、证据覆盖和可信度下降原因。

## 9. 修改招聘目标后重新评估

调整必备技能、加分技能或权重后，展示计划中的重新规划、按需复用、重新匹配、审查和排名变化。

## 10. 智能排期

展示候选人、面试官、会议室和时间冲突约束。确定性算法生成候选方案，Agent 只解释，HR 人工确认后才保存。

## 11. 结构化面试评价

面试官录入技术、项目、问题解决、沟通、风险和结论等结构化数据，供面试评估使用。

## 12. 简历与面试冲突审查

并列展示简历证据、面试证据、冲突说明、审查结论和可信度变化，不覆盖原始评分。

## 13. 企业招聘需求量化

展示目标人数、合格供给、高可信供给、尚缺人数、满足率和关键技能缺口；所有指标必须来自企业目标和当前数据库。

## 14. HR 最终报告

展示目标、排名、评分分解、证据、来源、面试状态、审查结论、可信度和下一步建议，最终决定由 HR 完成。

## 15. 员工小程序

最终目标只展示员工首页、签到签退、今日/本月考勤、年假、本人薪资摘要和简单制度查询，不进入招聘或 HR 后台。

## 16. Gradio 技术展示

Gradio 仅作为内部调试入口，计划展示节点、Tool、来源、耗时、错误与回退模式，不作为正式业务入口。

## 17. LLM/RAG 降级说明

LLM、LangGraph 或知识库不可用时，明确展示功能未接入或受控降级；确定性评分、排期、考勤、薪资预审和权限控制仍独立工作。不得用静态原型、固定数据、文档截图或假事件冒充功能成功。
