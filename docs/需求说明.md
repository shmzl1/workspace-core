# 需求说明

## 项目定位

TalentFlow 智聘中枢是面向招聘决策、员工服务、考勤薪资预审与权限审计的可解释企业人力资源管理 Agent。系统通过一套 FastAPI 后端支撑 Vue Web 管理端、微信小程序员工端和 Gradio 内部调试台。

## 角色

- 普通员工：查看个人考勤、年假、本人薪资摘要和制度问答。
- HR 专员：管理岗位、候选人、招聘流程、面试排期、薪资预审和审计。
- 部门主管：查看部门相关候选人、面试安排和招聘建议。
- 薪酬管理员：查看授权范围内的薪资预审、薪资访问记录和审计信息。

## Web 管理端功能

- 招聘驾驶舱、岗位、候选人、候选人评分、权重沙盘、候选人比较。
- 招聘流程看板、面试排期、排期冲突解释。
- 员工服务、制度查询、考勤汇总。
- HR 薪资预审、待 HR 确认、薪资权限、字段脱敏和审计。
- Agent Trace、招聘报告和风险提示。

## 小程序员工端功能

一期仅提供员工端简单能力：

- 员工首页。
- 签到、签退。
- 今日考勤状态。
- 本月考勤摘要。
- 年假余额。
- 本人薪资摘要。
- 制度查询。

## Agent 功能

“招聘决策 Agent”表示完整招聘工作流，不作为与正式节点并列的单体 Agent。正式架构为：

```text
企业招聘目标
→ 招聘策略 Agent
→ 简历解析 Agent / 岗位匹配 Agent / 面试评估 Agent
→ 决策审查 Agent
→ HR 最终报告
→ HR 人工决定
```

- 企业招聘目标：岗位、部门、目标人数、截止日期、必备/加分技能、经验、评分阈值、可信度阈值、紧急程度和可选薪资预算。
- 招聘策略 Agent：读取目标与候选人范围，生成招聘执行计划并分派专业节点，不直接生成最终评分。
- 简历解析 Agent：提取候选人画像、缺失项和可定位简历证据，不负责最终岗位评分。
- 岗位匹配 Agent：读取画像、岗位 Rubric、必备条件和企业知识来源，经 Service 使用确定性评分，输出匹配与技能缺口。
- 面试评估 Agent：只使用真实结构化面试数据；无数据时保持待面试或 `SKIPPED`。
- 决策审查 Agent：检查无证据高分、必备条件、来源版本、简历与面试冲突及节点分歧，不静默修改确定性评分。
- HR 最终报告：汇总排名、证据、知识来源、面试状态、可信度和建议；录用、淘汰、排期确认和薪资确认仍由 HR 人工决定。
- 员工服务 Agent：查询本人考勤、年假、本人薪资摘要和制度，不属于招聘主工作流。
- 薪资预审助手：解释预审结果、提示异常和生成审查建议，不属于招聘主工作流。

招聘体验还包括企业需求量化、候选人比较、可信度分解、实时执行状态，以及修改企业招聘目标后的重新评估。当前相关状态必须区分“代码存在，待本地人工验收”“已建立目录或契约”和“计划中”。

## 考勤范围

- 仅做时间签到签退。
- 固定状态：正常、迟到、早退、缺勤、无薪请假、已批准年假。
- 暂时不做 GPS、Wi-Fi、人脸、蓝牙等位置考勤。

## 薪资预审范围

- 规则引擎负责计算。
- AI 负责异常解释和审查建议。
- HR 负责最终确认。
- Agent 不得确认工资、修改工资、删除扣款或写入已确认薪资。

## Sprint 1 黄钧外层范围

- 招聘评分外层：展示岗位、候选人、申请记录和评分入口；真实评分依赖黄钧人工维护的 `score_resume(...)`。
- 智能排期外层：展示候选人、面试官、会议室、时间槽和排期预览入口；真实排期依赖黄钧人工维护的 `schedule_interview(...)`。
- HR 薪资预审展示外层：展示预审记录、扣款来源、规则说明和“待 HR 确认”状态；Sprint 1 不做最终薪资确认。
- 禁飞区未接入时，系统展示待人工接入说明，不使用 mock 算法冒充真实结果。

## 非功能需求

- 结果可解释。
- 权限控制和字段脱敏。
- 敏感操作审计。
- 局域网部署演示。
- 使用演示数据，不使用真实企业数据。

## 不做项

- 不做第二套后端、微服务、Redis、Celery、RabbitMQ 或 Kubernetes。
- 不做真实微信 OAuth、公网生产部署、真实企业数据接入。
- 小程序不做招聘、候选人、面试排期、薪资预审或审计后台。
- 薪资模块不做银行发薪、税务、社保、公积金。
- Agent 不自动确认工资，不绕过人工禁飞区。
