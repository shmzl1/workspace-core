# 测试计划

本文记录规划中的测试范围，不表示测试已经执行或通过。

## 测试范围

- 禁飞区人工测试：由人工负责人编写和解释核心算法测试。
- 数据库迁移检查：后续人工执行 Alembic 升级后，确认 22 张初始表和约束能在 PostgreSQL 中创建。
- 单元测试：覆盖 Service、Repository、权限判断和规则计算外层编排。
- API 集成测试：覆盖统一响应、错误返回、身份上下文和权限边界。
- 前端页面状态测试：覆盖加载、空、错误、权限拒绝、Toast、表单和 Agent Trace 展示。
- Agent Tool 调用测试：覆盖 Tool 到 Service 的调用链。
- RAG 来源测试：覆盖制度命中片段和来源展示。
- 权限与脱敏测试：覆盖薪资字段可见范围和拒绝访问。
- 薪资预审状态机测试：覆盖草稿、已生成预审、待 HR 确认、已确认。
- Web 与小程序数据一致性测试：覆盖员工考勤、年假、本人薪资摘要。

## Sprint 2.1 招聘策略运行验收

以下为待人工执行的验收清单，不表示已经运行或通过：

- 使用 `agent.hr.use` 账号创建 Run 并获得真实 `run_id`。
- 招聘策略节点经历 `WAITING -> RUNNING -> COMPLETED`。
- SSE 按顺序收到六类真实事件，并能重放历史事件。
- `AGENT_THINKING` 仅包含结构化阶段摘要。
- 其他五个节点为 `SKIPPED`，原因是 `CURRENT_PHASE_NOT_IMPLEMENTED`。
- URL 中的 `run_id` 可用于页面刷新恢复；后端重启后显示进程内记录已丢失。
- 不同用户不能读取彼此 Run；无权限账号不能调用接口。
- 页面不存在假 Tool、假 RAG、假评分、假面试数据、随机日志或固定延迟。
- Runtime 不访问 Repository 或 `human_only`，数据库 Session 不进入后台 Runner。
- Snapshot 新增 `sources` 后，现有 Run 创建不需要额外参数，前后端仍保留 `created_at`、`updated_at` 和执行计划阶段字段。
- 未来候选人、岗位匹配、面试评价、决策审查、HR 报告与 RAG 类型只做契约导入检查，不把契约存在误报为真实节点或检索已运行。
