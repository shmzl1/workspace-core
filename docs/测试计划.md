# 测试计划

本文记录规划中的测试范围，不表示测试已经执行或通过。

## 测试范围

- 禁飞区人工测试：由人工负责人编写和解释核心算法测试。
- 数据库迁移检查：后续人工执行 Alembic 升级后，确认 22 张初始表和约束能在 PostgreSQL 中创建。
- 单元测试：覆盖 Service、Repository、权限判断和规则计算外层编排。
- API 集成测试：覆盖统一响应、错误返回、身份上下文和权限边界。
- 前端页面状态测试：覆盖加载、空、错误、权限拒绝、Toast、表单和 Agent Trace 展示。
- Agent Tool 调用测试：覆盖 Tool 到 Service 的调用链。
- RAG 来源测试：覆盖制度命中片段和来源展示。
- 权限与脱敏测试：覆盖薪资字段可见范围和拒绝访问。
- 薪资预审状态机测试：覆盖草稿、已生成预审、待 HR 确认、已确认。
- Web 与小程序数据一致性测试：覆盖员工考勤、年假、本人薪资摘要。

## Sprint 质量活动

- Sprint 1：Bug 注入与基础回归，重点检查招聘评分外层、排期外层、签到签退、本人薪资查询、权限拒绝和审计。
- Sprint 2：即兴修改与回归，重点检查 Agent、RAG、薪资预审解释、待 HR 确认和候选人比较。
- Sprint 3：演示路径回归，重点检查 Web、小程序、局域网访问、Docker 计划配置和演示数据。

## Sprint 1 黄钧手动验收项

- 后端外层检查：Swagger 中能看到招聘、面试排期、薪资预审相关接口；接口响应保持 `success`、`data`、`error`、`trace_id` 结构。
- 招聘评分外层：调用评分入口时，若禁飞区未接入，应返回 `HUMAN_ONLY_ALGORITHM_NOT_READY` 或等价说明。
- 面试排期外层：调用排期预览时，若禁飞区未接入，应展示排期算法待黄钧人工接入和结构化冲突说明。
- HR 薪资预审展示：页面展示预审记录、扣款来源、规则说明和“待 HR 确认”状态，不提供真正确认工资能力。
- 前端页面检查：`Sprint 1 闭环` 三个 Tab 均可展示加载后的业务状态或后端待联调提示。
- 禁飞区检查：确认 AI 未创建或修改 `resume_scoring.py`、`interview_scheduler.py` 及其核心测试。

以上为手动验收清单，不表示本次已经运行 pytest、前端构建、后端启动或接口联调。
