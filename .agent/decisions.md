# TalentFlow 智聘中枢技术决策记录

## ADR-001：正式主界面选择 Vue 3，而不是 Gradio

- 状态：已接受
- 日期：2026-07-06
- 决策：正式主界面使用 Vue 3 + TypeScript + Vite。
- 原因：项目需要企业 SaaS 工作台，包括模块导航、全局搜索、通知、主题切换、角色切换、复杂表格、图表、日历、看板和权限化页面。Vue 3 更适合正式业务界面。
- 影响或后果：正式业务入口由 Vue 承担；Gradio 不用于正式用户操作。

## ADR-002：Gradio 保留为内部 Agent 调试台

- 状态：已接受
- 日期：2026-07-06
- 决策：保留 Gradio，仅用于内部 Agent 调试和备用演示。
- 原因：Gradio 可以快速查看 LangGraph 执行链、工具调用、RAG 命中内容和错误信息，适合调试 Agent。
- 影响或后果：Gradio 不承担正式权限、正式流程和正式数据管理入口。

## ADR-003：后端选择 FastAPI + Python，而不是 SpringBoot + Python 双后端

- 状态：已接受
- 日期：2026-07-06
- 决策：后端统一使用 Python 3.12 + FastAPI。
- 原因：Agent、RAG、LangGraph、LangChain Tools、ChromaDB 和 OpenAI 兼容模型接口都在 Python 生态中集成更直接。双后端会增加接口联调、部署和鉴权复杂度。
- 影响或后果：所有业务 API、Agent 编排和 RAG 能力集中在一个 FastAPI 应用中。

## ADR-004：采用模块化单体，不采用微服务

- 状态：已接受
- 日期：2026-07-06
- 决策：后端采用 FastAPI 模块化单体。
- 原因：团队规模为 4 人，项目周期为 12 天，模块化单体部署简单、调试简单、边界清晰。微服务会引入额外的服务治理、跨服务通信和部署成本。
- 影响或后果：通过模块目录和层次约束维护边界，不引入服务拆分。

## ADR-005：使用 LangGraph 组织员工服务 Agent 与招聘决策 Agent

- 状态：已接受
- 日期：2026-07-06
- 决策：使用 LangGraph 编排员工服务 Agent 与招聘决策 Agent。
- 原因：员工服务和招聘决策涉及自然语言、多工具调用、RAG、权限判断和跨模块任务。LangGraph 适合显式建模执行节点、工具调用和状态流转。
- 影响或后果：Agent 逻辑放入后端 `agents` 模块，工具能力放入 `tools` 模块。

## ADR-006：使用 PostgreSQL 保存业务数据

- 状态：已接受
- 日期：2026-07-06
- 决策：使用 PostgreSQL 保存结构化业务数据。
- 原因：员工、岗位、候选人、招聘流程、面试排期、薪资权限、审计日志和通知都属于结构化数据，PostgreSQL 能提供稳定事务和关系建模能力。
- 影响或后果：后端通过 SQLAlchemy 和 Alembic 管理数据访问与迁移。

## ADR-007：使用 ChromaDB 建立企业制度知识库 RAG

- 状态：已接受
- 日期：2026-07-06
- 决策：使用 ChromaDB 存储和检索企业制度知识库向量。
- 原因：员工制度问答需要检索请假、报销、薪资保密、试用期、招聘等制度，并展示来源。ChromaDB 适合本地部署和快速集成。
- 影响或后果：RAG 层需要返回命中片段和来源信息，回答不得只给结论。

## ADR-008：AI 禁飞区保持为三个独立的纯 Python 文件，不拆成多个服务，也不让 AI 生成实现

- 状态：已接受
- 日期：2026-07-06
- 决策：禁飞区只包含 `resume_scoring.py`、`interview_scheduler.py`、`salary_access_control.py` 三个纯 Python 文件，由人工手写核心算法。
- 原因：候选人评分、面试排期和薪资权限是项目最核心、最需要可解释和可控的算法逻辑。保持单文件、单入口、纯函数式边界有利于审查和测试。
- 影响或后果：AI 不得生成或修改禁飞区实现。Agent 只能通过 `Tool -> Service -> human_only` 调用核心算法。

## ADR-009：正式演示以本地 Docker Compose 部署为目标，降低外网和环境风险

- 状态：已接受
- 日期：2026-07-06
- 决策：正式演示目标为本地 Docker Compose + Nginx 部署。
- 原因：本地部署能降低外网、云资源、账号权限和环境差异风险，适合课程项目和现场演示。
- 影响或后果：部署设计围绕本地容器编排；不引入 Kubernetes。

## ADR-010：Agent 只处理自然语言、多工具、跨模块任务；普通确定性操作走普通业务 API

- 状态：已接受
- 日期：2026-07-06
- 决策：普通 CRUD 和确定性业务操作走 FastAPI 业务 API；自然语言、多工具、跨模块任务才走 Agent。
- 原因：Agent 适合处理不确定输入和工具编排，但普通操作使用 API 更稳定、可测、可审计。
- 影响或后果：前端需要区分普通页面操作和 Agent 指令；后端需要保持 API、Service、Agent Tool 三类入口边界清晰。
